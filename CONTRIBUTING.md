# Contributing

Contributions are welcome, and they are greatly appreciated! Every bit helps, and credit will always be given.

You can contribute in many ways:

---

## Types of Contributions

### Report Bugs

Report bugs at [GitHub Issues](https://github.com/genuinemerit/saskan-lore/issues).

When reporting a bug, please include:

- Your operating system name and version.
- Any details about your local setup that might be helpful in troubleshooting.
- Detailed steps to reproduce the bug.

### Implement Features

Check open issues tagged with `enhancement` and `help wanted`. These are open for community implementation.

### Write Documentation

The project wiki (`saskan-wiki`) and in-repo docs always benefit from improvement — whether developer-facing or user-facing.

### Submit Feedback

The best way to send feedback is through [GitHub Issues](https://github.com/genuinemerit/saskan-lore/issues).

If proposing a feature:

- Explain in detail how it would work.
- Keep the scope as narrow as possible to simplify implementation.
- Remember this is a volunteer-driven project.

---

## Pull Request Process

1. Fork the repo and create your branch from `main`:

   ```bash
   git checkout -b feature/your-feature-name
   ```

2. If you’ve added code, add or update tests as needed

3. Ensure the test suite passes

4. Ensure your code follows lint/format rules (see project configs)

5. Update documentation if applicable

6. Open a Pull Request with a clear description of your changes

---

## Coding Standards

Follow PEP 8 style guidelines for Python

Run formatters/linters (black, isort, flake8) before committing

Keep commits atomic and messages descriptive

---

## Code of Conduct

This project follows the Contributor Covenant (CODE_OF_CONDUCT.md). By participating, you are expected to uphold this standard.
