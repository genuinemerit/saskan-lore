# NFR-001: Local-First

## Requirement

The system shall run entirely on local hardware with no dependency on external cloud APIs
for core pipeline operations.

## Rationale

Local operation ensures data privacy, works offline, incurs no ongoing API costs, and keeps
the full stack inspectable. See ADR-001 (no training) and ADR-002 (SQLite).

## Implications

- SQLite is the only database backend; no remote database connections.
- Model inference uses a local GGUF model via llama.cpp; no OpenAI or similar API is required
  for extraction or answering.
- Two workstations are in use: a Dell Linux laptop (primary dev/test) and an Apple M2 Pro Mac
  (target runtime, Metal GPU acceleration). See ADR-008 for environment configuration and
  model selection by machine. See the Glossary entry for M2 Mac vs. Dell Linux Laptop.
- External API calls (e.g., for embeddings) are permissible only if explicitly decided in
  a new ADR.
