# Non-Functional Requirements

One file per requirement. Each defines a quality constraint on how the system shall behave, with rationale and implications.

| NFR | Title | Notes |
| --- | --- | --- |
| [NFR-001](nfr_001.md) | Local-First | No cloud API dependency for core operations |
| [NFR-002](nfr_002.md) | Replaceable Model Layer | Inference decoupled from data and retrieval |
| [NFR-003](nfr_003.md) | Traceability | Every claim linked to chunk and verbatim source span |
| [NFR-004](nfr_004.md) | Human Review Gate | No unreviewed claim used for retrieval or evaluation |
| [NFR-005](nfr_005.md) | Pilot Pragmatism | Favor simplicity; keep all artifacts inspectable |
