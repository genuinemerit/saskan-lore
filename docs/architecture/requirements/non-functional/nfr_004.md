# NFR-004: Human Review Gate

## Requirement

No claim shall be used for retrieval or evaluation until it has been marked as human-reviewed.

## Rationale

LLM extraction output is untrusted by default. The review gate is the control point that prevents hallucinated, misclassified, or imprecise claims from entering the trusted corpus. See Workflow Stage 4 and ADR-007.

## Implications

- Claims are stored with `reviewed = false` immediately after extraction.
- Only claims with `reviewed = true` are eligible for retrieval (FR-006) or evaluation (FR-008).
- Batch loads must not set `reviewed = true` automatically; the flag requires explicit human action.
- Rejected claims should be retained with a `rejected` status rather than deleted, for audit purposes.
