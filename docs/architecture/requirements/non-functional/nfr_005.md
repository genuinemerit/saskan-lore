# NFR-005: Pilot Pragmatism

## Requirement

The design may favor simplicity and speed of learning over long-term elegance, provided all artifacts remain inspectable and reusable.

## Rationale

Over-engineering at this stage wastes effort without confirming that the core pipeline
works. The pilot validates the full pipeline end-to-end before scaling. See ADR-006
(graduation criteria).

## Implications

- Skipping abstractions that are not yet needed is acceptable.
- Skipping documentation, leaving the schema undocumented, or producing data in an
  uninspectable state is not acceptable.
- Code must be readable and runnable, even if not fully generalized.
- Decisions that trade elegance for speed must still be recorded (as ADRs if
  significant, as inline comments if minor).
