# NFR-002: Replaceable Model Layer

## Requirement

The model and inference layer shall be kept separate from the data model, retrieval logic, and storage layer.

## Rationale

Model choice is likely to change — from a 3B local GGUF to a larger model, or from llama.cpp to a hosted API. Decoupling inference from the rest of the system means that change does not require rewriting retrieval or storage code. See ADR-001.

## Implications

- Inference is called through a defined interface; no model-specific types, formats, or behaviors leak into the data or retrieval layers.
- Swapping the model requires only changes to the inference module, not to chunking, claim storage, or retrieval.
- Prompt templates are stored separately from inference code and from data access code.
