# FR-008: Evaluation

## Requirement

The system shall store evaluation questions with expected answers, record model answers and pass/fail results for each evaluation run, and support failure analysis.

## Details

This FR covers both the evaluation question set and the per-run result records.

**Evaluation questions** (`eval_questions` table):

| Field | Description |
| --- | --- |
| `question_id` | stable unique identifier |
| `question_text` | the question posed to the system |
| `expected_answer` | the correct answer, as determined by human review |
| `scope` | lore domain this question covers (e.g., `"varkaar"`) |

**Evaluation results** (`eval_results` table):

| Field | Description |
| --- | --- |
| `result_id` | stable unique identifier |
| `question_id` | the question evaluated |
| `model_answer` | the answer generated by the system |
| `retrieved_evidence` | the chunks/claims used to generate the answer |
| `pass_fail` | `pass` or `fail` |
| `failure_type` | if fail: one of `wrong_fact`, `hallucination`, `incomplete`, `style` |
| `notes` | free-text notes for failure analysis |
| `run_at` | timestamp of evaluation run |

Pilot target: 10 evaluation questions covering the Covenant of Varkaar domain.

## Example

```txt
question_id:        q_003
question_text:      "What punishment did the Covenant prescribe for oath-breaking?"
expected_answer:    "Death, under Covenant law."

result_id:          res_012
question_id:        q_003
model_answer:       "The Covenant prescribed death for oath-breaking."
retrieved_evidence: [claim_0091]
pass_fail:          pass
failure_type:       null
notes:              null
run_at:             2026-04-01T14:22:00
```
