# FR-006: Retrieval

## Requirement

The system shall retrieve relevant chunks and/or claims for a given user query and return them with source references.

## Details

- MVP retrieval uses keyword search or embedding + cosine similarity in Python. No external vector database. See ADR-002.
- Returns the top N results (default: 3). N is configurable.
- Each result includes: chunk or claim text, document title, chunk sequence number, and truth status (for claims).
- Only reviewed claims (`reviewed = true`) are eligible for retrieval.
- Retrieval must be scope-aware: pilot queries are limited to Varkaar documents.

## Example

```txt
Query:   "oath-breaking punishment"

Result 1:
  chunk_id:    chunk_0042
  document:    "Saskan Canon — Varkaar Covenant"
  text:        "The Covenant of Varkaar governed oath law..."
  claim:       "The Covenant regarded oath-breaking as a capital offense."
  truth_status: fact
```
