# FR-004: Claims

## Requirement

The system shall store extracted claims linked to source chunks and source spans, each carrying a truth-status label.

## Details

A claim is the primary unit of meaning. Every claim must be traceable to a specific verbatim passage in the source. See ADR-003 and ADR-004.

| Field | Description |
| --- | --- |
| `claim_id` | stable unique identifier |
| `chunk_id` | chunk the claim was extracted from |
| `document_id` | parent document (denormalized for query convenience) |
| `claim_text` | the extracted statement, in clean prose |
| `source_span` | verbatim quote from the source that supports this claim |
| `truth_status` | one of: `fact`, `belief`, `interpretation`, `rumor` |
| `reviewed` | boolean — true only after human review |

Truth-status definitions:

| Status | Meaning |
| --- | --- |
| `fact` | stated as objective truth in the source |
| `belief` | held as true by a character or faction, not the narrator |
| `interpretation` | an inference drawn from evidence, not directly stated |
| `rumor` | reported as unverified or hearsay within the source |

Only claims with `reviewed = true` may be used for retrieval or evaluation.

## Example

```txt
claim_id:    claim_0091
chunk_id:    chunk_0042
document_id: doc_001
claim_text:  "The Covenant of Varkaar regarded oath-breaking as a capital offense."
source_span: "...oath-breakers were put to death under Covenant law..."
truth_status: fact
reviewed:    true
```
