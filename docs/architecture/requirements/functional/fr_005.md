# FR-005: Relationship Storage

## Requirement

The system shall store typed, directed relationships between entities.

## Details

- Relationships are stored as directed edges: a source entity, a target entity, and a relationship type.
- Both entities must exist in the entity catalog before a relationship can be inserted.
- Bidirectional relationships are stored as two separate directed records.
- Relationship types use a controlled vocabulary (e.g., `governs`, `allied_with`, `opposed_to`, `member_of`). New types may be added but must be documented.
- Each relationship should be traceable to at least one supporting claim.

| Field | Description |
| --- | --- |
| `relationship_id` | stable unique identifier |
| `source_entity_id` | the entity the relationship originates from |
| `target_entity_id` | the entity the relationship points to |
| `relationship_type` | string from controlled vocabulary |
| `claim_id` | claim that supports this relationship (nullable) |

## Example

```txt
relationship_id:   rel_005
source_entity_id:  ent_017   # Covenant of Varkaar
target_entity_id:  ent_003   # Northern Provinces
relationship_type: governs
claim_id:          claim_0091
```
