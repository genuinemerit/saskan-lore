# FR-003: Entity Catalog

## Requirement

The system shall store named entities with stable IDs and support one or more alias names per entity.

## Details

- Entities represent discrete named things in the lore: persons, places, factions, artifacts, eras, and similar.
- Each entity has a canonical name and an entity type. Aliases are stored in a separate `entity_aliases` table linked by `entity_id`.
- Entity IDs must be stable once assigned. An entity record must never be deleted or have its ID changed; it may be deprecated.
- Canonical names should be normalized to a consistent form (e.g., title case) at ingestion time.

| Field | Description |
| --- | --- |
| `entity_id` | stable unique identifier |
| `canonical_name` | primary name used for display and matching |
| `entity_type` | one of: `person`, `place`, `faction`, `artifact`, `era`, `other` |

## Example

```txt
entity_id:      ent_017
canonical_name: "Covenant of Varkaar"
entity_type:    faction

aliases: ["the Covenant", "Varkaar Covenant", "the northern Covenant"]
```
