# FR-002: Chunk Storage

## Requirement

The system shall split each source document into retrievable text chunks and store them linked to their parent document.

## Details

- Chunks preserve source order and are assigned a sequence number within the document.
- Chunk text is stored verbatim except for approved normalization (e.g., whitespace collapsing, Unicode normalization).
- Chunk boundaries must be reproducible: re-running the chunker on the same input must produce the same chunks.
- Each chunk record links to its parent `document_id`.

| Field | Description |
| --- | --- |
| `chunk_id` | stable unique identifier |
| `document_id` | parent document |
| `sequence` | position within the document (0-indexed) |
| `text` | verbatim chunk content |

## Example

```txt
chunk_id:    chunk_0042
document_id: doc_001
sequence:    42
text:        "The Covenant of Varkaar governed oath law in the northern
              provinces during the Ashen Era."
```
