# ADR-002: SQLite as System of Record

Status: Accepted

## Decision

SQLite is the only persistence technology in the pre-pilot. All structured data — documents, chunks, entities, claims, relationships, and evaluation records — is stored in a single SQLite database file. SQLAlchemy is used as the ORM layer for both DDL (schema definition) and DML (queries and writes).

## Why

SQLite is the simplest viable option for a local, single-user corpus of this scale. It eliminates server infrastructure, minimizes dependencies, and keeps the full data store inspectable as a single file. SQLAlchemy enforces good design discipline (explicit models, relationship declarations, session management) and makes a future backend migration feasible without rewriting data access code.

## Implications

- All data access goes through SQLAlchemy; raw SQL is avoided except for diagnostics.
- Schema is defined using SQLAlchemy declarative models, which serve as the authoritative source of truth for table structure.
- Schema changes require explicit Alembic migration scripts.
- SQLAlchemy's database-agnostic abstractions must be used throughout; no SQLite-specific types or behaviors in critical paths.
- No vector database will be introduced in the pre-pilot; embedding-based retrieval, if used, will be handled in Python.
