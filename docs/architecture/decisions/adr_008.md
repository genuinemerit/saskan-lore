# ADR-008: Multi-Environment Configuration and Platform-Aware Model Selection

Status: Accepted

## Decision

The project is developed and tested on a Dell Linux laptop (Ubuntu, primary dev workstation)
and run on an Apple M2 Pro Mac (music studio, target inference machine). Each environment
uses a machine-local `env.local` file (gitignored) to configure environment-specific values.
The inference layer selects model path and GPU settings from these env vars at startup — no
hardcoded paths or platform detection logic in application code.

## Environments

| Environment | Machine | Role | Inference backend |
| --- | --- | --- | --- |
| `development` | Dell laptop, Ubuntu, Intel/AMD CPU | Dev and test | llama.cpp CPU-only |
| `production` | Apple M2 Pro, macOS, Metal GPU | Target runtime | llama.cpp + Metal |

## Model storage

GGUF model files are stored at `~/models/` on each machine. This path is outside any
cloud-synced directory (e.g. Dropbox) to prevent large binary files from syncing and to
avoid risk of Smart Sync eviction during inference (which would stall `mmap`).

`~/models/` is consistent across both machines: `/home/dave/models/` on Linux,
`/Users/dave/models/` on Mac. `LOCAL_MODEL_PATH` in `env.local` holds the absolute path
to the specific GGUF file in use on that machine.

## Model selection by environment

The M2 Pro has sufficient unified memory to run a 7B-class model at Q4_K_M quantization
with full Metal GPU offload. The Dell laptop runs the same or a 3B-class model CPU-only,
which is slower but correct.

| Machine | Model | Quantization | `LLAMA_N_GPU_LAYERS` |
| --- | --- | --- | --- |
| M2 Pro (Mac) | Qwen2.5-7B-Instruct | Q4_K_M | -1 (all layers to Metal) |
| Dell (Linux) | Qwen2.5-3B-Instruct | Q4_K_M | 0 (CPU only) |

`-1` for `LLAMA_N_GPU_LAYERS` means offload all layers to the Metal GPU. `0` means CPU only.
The inference module reads this value at startup; no platform detection logic is needed in code.

## Why env vars over platform detection

Platform detection (`platform.system()`, checking for Metal) in application code creates
hidden branches that are hard to test and easy to get wrong. Explicit env vars are visible,
overridable, and documented. Each machine's `env.local` is the authoritative statement of
what that machine is configured to do.

## Implications

- `env.example` documents all required env vars with comments distinguishing Mac vs Linux values.
- `env.local` is never committed; each workstation maintains its own copy.
- The inference module must read `LOCAL_MODEL_PATH` and `LLAMA_N_GPU_LAYERS` from the
  environment — no defaults that silently work on one machine and fail on another.
- A missing or empty `LOCAL_MODEL_PATH` must raise a clear error at startup, not at
  first inference call.
- Model files are never stored inside the project directory or any cloud-synced path.
