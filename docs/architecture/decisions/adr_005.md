# ADR-005: Relationships Modeled from the Start

Status: Accepted

## Decision

The schema will include an explicit relationships table from the start, linking named entities by typed, directed relationships. This table may be sparsely populated in the pre-pilot but must be structurally present.

## Why

Retrofitting relationship support into an existing schema is costly and risks data integrity issues. The pre-pilot is the right time to establish the structure, even if only a small number of relationships are loaded initially.

## Implications

- The relationships table requires both entities to exist in the entities table before a relationship can be inserted (referential integrity enforced).
- Entity IDs must be stable; updates to entity records must not change their primary key.
- Relationship types are strings (e.g., `"governs"`, `"allied_with"`, `"opposed_to"`); a controlled vocabulary for types will be defined as part of the schema.
- Bidirectional relationships are stored as two directed records, not a single undirected one.
